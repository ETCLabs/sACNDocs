◆ source_count_max

int SacnMergeReceiverConfig::source_count_max

The maximum number of sources this universe will listen to.

May be kSacnReceiverInfiniteSources. This parameter is ignored when configured to use static memory – the lower of SACN_DMX_MERGER_MAX_SOURCES_PER_MERGER or SACN_RECEIVER_MAX_SOURCES_PER_UNIVERSE is used instead.

◆ use_pap

bool SacnMergeReceiverConfig::use_pap

If true, this allows per-address priorities (if any are received) to be fed into the merger.

If false, received per-address priorities are ignored, and only universe priorities are used in the merger. Keep in mind that this setting will be ignored if SACN_ETC_PRIORITY_EXTENSION = 0, in which case per-address priorities are ignored.
